Quick Start

Last updated: Jan 25th, 2016

Introduction

This tool is intended for testing the PolyA-1 prototype for Knee Internal/External Rotation in conjunction with Ankle Flexion/Extion.

The assumption is that a Control Dataset output from a fabricated path given by the PolyA-1 goniometer readings will be used to determine how close a Test Subject Data is to such path when given free range of motion while using the machine.

Data Input

Step One

To begin the test, record the readings for the Control Dataset from the PolyA-1 goniometer reading/calibration software to a new file.

Next, copy/paste the readings as they are outputted by the PolyA-1 goniometer readings in the Control Dataset text box.


The graph will reflect the inputted readings like so:

Step Two

The control data is now set in memory and is ready to be tested against the Test Subject Data.

Note

Be sure to save goniometer readings to a new file every time a new test run is made with the PolyA-1 goniometer reading/calibration software.

Be sure to calibrate the the PolyA-1 goniometer readings and let the Test Subject use the machine with free range of motion.

Instruct the test subject to follow the path from step 1. (It helps to have a drawn image of the Control Path in front of the machine and having the test subject follow along with a lazer pointer).

Finally, copy/paste the new readings to the Test Subject Dataset input box:


The application will begin analyzing the data as soon as the data is pasted into the box. And after a breif loading period the graph will super-impose the new readings against the old as well as provide analitical and quatitative results from the two given datasets in comparison.

Step Three

This application allows to set a level of tolerance when calculating the accuracy of the test subject path's points against the control path. A default setting of 1.0 is given.


The tolerance metric treated as the allowed distance from two points form each path. If the distance between the points is greater than the tolerance specified, the test subject's path is said to be "outside" of the desired path, and vise-versa.

Accuracy percentages and deviations are calculated with based on this tolerance. The smaller the tolerance, the more precise the analysis will be, meaning there is less margin for error.

Data Analysis

Points Inside/Outside

Total points from the Test Subject Dataset that were found to be "inside" the tolerance range from the given Control Dataset.

Error Spread

The farthest distance that the test subject's path was from the control path.

This is calculated for every point on the Test Subject Dataset against all points in the same quadrant from the Control Dataset
