Quick Start

Last updated: Jan 25th, 2016

Introduction

The PolyA-1 prototype outputs synchronized path readings combined with analog-digital readings. These readings are comprised of Knee Internal/External Rotation, Ankle Flexion/Extention, and Inversion/Evertion in degrees with more than 3 digits.

This application - for the sake of simplicity and the ability to test the PolyA-1 system's reliability and usability - is focused solely on two of these data readings: Knee Internal/External Rotation in conjunction with Ankle Flexion/Extention. These two readings are enough to spacially observe the range of motion in a 2D setting. For the purpose of the study, the third axis of motion (Inversion/Evertion) is temporarily locked.

The Control Dataset is simply the output readings that the machine gives out when it is moved mechanically by a motor while following a pre-determined path. These readings are to be considered the exact path the test subject must output when effectively following the same path given free range of motion of the involved axis.

This is achieved by instructing a test subject to "follow" a path by moving the foot (along with the machine strapped to the ankle and foot) and using a pointer as reference to where the foot is pointing giving real-time visual feedback of the path trajectory.

The most integral and simple path chosen for this study could be a circle. Other basic paths to test centralized vectors of motion generated by combination of the two axis can be horizontal/vertical or even diagonal lines.

The two axis of motion aforementioned are then compared graphically and quantatatively with a given tolerance for error.

Instructions

Step One

To begin the test, record the readings for the Control Dataset from the PolyA-1 goniometer reading/calibration software to a new file.

Next, copy/paste the readings as they are outputted by the PolyA-1 goniometer readings in the Control Dataset text box.

Note

Since the goniometer readings are much more precise than needed, the readings reflect a "noise" that can be observed in the graph above. The circle path above is to be considered a perfect circle as far as human error is concerned. This is the reason a level tolerance is used when analyzing data with such precise readings.

Step Two

The control data is now set in memory and is ready to be tested against the Test Subject Data.

Note

Be sure to save goniometer readings to a new file every time a new test run is made with the PolyA-1 goniometer reading/calibration software.

Be sure to calibrate the the PolyA-1 goniometer readings and let the Test Subject use the machine with free range of motion.

Instruct the test subject to follow the path from step 1. (It helps to have a drawn image of the Control Path in front of the machine and having the test subject follow along with a lazer pointer).

Finally, copy/paste the new readings to the Test Subject Dataset input box:


The application will begin analyzing the data as soon as the data is pasted into the box. And after a breif loading period the graph will super-impose the new readings against the old as well as provide analytical data from the two given datasets in comparison.

Step Three

This application allows to set a level of tolerance when calculating the accuracy of the test subject path's points against the control path. A default setting of 1.0 is given.


The tolerance metric treated as the allowed distance from two points form each path. If the distance between the points is greater than the tolerance specified, the test subject's path is said to be "outside" of the desired path, and vise-versa.

Accuracy percentages and deviations are calculated with based on this tolerance. The smaller the tolerance, the more precise the analysis will be, meaning there is less margin for error.

Data Analysis

Points Inside/Outside

Total points from the Test Subject Dataset that were found to be "inside" the tolerance range from the given Control Dataset.

Error Spread

The farthest distance that the test subject's path was from the control path.

This is calculated for every point on the Test Subject Dataset against all points in the same quadrant from the Control Dataset

Path Coverage

The percentage of coverage for all 4 coverage areas determined by the 4 quadrants of the dataset.

The test subject dataset is considered to cover a quadrant when a given poin (Knee Internal/External Rotation, Ankle Flexion/Extention) is found inside the quadrant.

A 100% coverage sugests the test subject has reached each of the 4 sectioned areas for a given path.

Quadrant Accuracy

Each quadrant is given a percentage of accuracy for the test subject path compared to the control path in the corresponding quadrant. Where accuracy is the percentage of points considred to be "inside" the correct path given the allowed tolerance for error.

A 100% accuracy sugests the test subject has effectively stayed within the tolerance of the desired path for the entire quadrant.

Overall Positional Accuracy

The total percentage of points considred to be "inside" the correct path given the allowed tolerance for error for the entire desired path.

A 100% accuracy sugests the test subject has effectively stayed within the tolerance of the desired path for the entire test.
